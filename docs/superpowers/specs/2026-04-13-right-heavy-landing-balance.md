# Right Heavy Landing Balance

Date: 2026-04-13

## Goal

Reduce the desktop left panel and give stronger visual focus to the right operational panel.

## Approved Direction

Use `right-heavy split`.

## Problem

The desktop layout is cleaner now, but the left side still takes more attention than needed.

The user wants:
- less emphasis on the left panel
- more focus on the right verification panel

## Scope

Refine the landing page at `/`.

Do not change:
- mobile layout direction
- phone verification behavior
- quick access destinations
- install prompt logic
- app branding

## Layout Direction

### Desktop

Desktop should become more right-heavy:

- left side:
  - narrow brand rail
  - logo
  - product name
  - one short descriptor
  - one shorter headline
- right side:
  - dominant verification panel
  - quick access
  - install surface

The right side should feel like the real workspace entry point.

### Mobile

Keep the current mobile-native layout unchanged.

## Visual Changes

### Left Side

- reduce width
- reduce copy volume
- reduce visual dominance
- keep clean branding

### Right Side

- increase relative width
- keep verification strongest
- give operational UI more visual authority

## Success Criteria

- desktop feels more product-like
- left side acts as support, not hero
- right side becomes the clear focus
- mobile remains unchanged
